## 一、项目概述

### 1.1 项目定位

「法拍者联盟」是一个专注于上海、宁波两地法院不动产司法拍卖信息聚合与服务的微信小程序平台。平台聚合淘宝法拍（sf.taobao.com）、京东法拍（pmsearch.jd.com）、公拍网（s.gpai.net）三大司法拍卖平台的房源数据，为用户提供一站式法拍房信息浏览、筛选、收藏和购房需求对接服务。

### 1.2 核心模块

| 模块 | 说明 |
|------|------|
| **微信小程序** | C端用户产品，法拍房浏览/搜索/收藏/选房需求提交 |
| **管理后台（Web）** | B端运营系统，房源管理/用户管理/需求分配/内容管理 |
| **数据爬虫** | 自动化数据采集，三大平台15个数据源每日更新 |
| **API后端** | Python FastAPI，为小程序和管理后台提供统一接口 |

### 1.3 目标城市

- **上海市**（city_id = 310000）
- **宁波市**（city_id = 330200）

### 1.4 数据来源

| 平台 | 搜索URL数 | 爬取方式 |
|------|-----------|---------|
| 淘宝司法拍卖（sf.taobao.com） | 10个（2城市 × 5物业类型） | Playwright无头浏览器 |
| 京东拍卖（pmsearch.jd.com） | 1个 | Playwright无头浏览器 |
| 公拍网（s.gpai.net） | 4个（2城市 × 2资产类型） | httpx优先，Playwright回退 |

## 二、UI设计方案

### 2.1 设计风格

**「莫兰迪蓝优雅风 V2」** — 基于UI.html中确定的设计令牌体系：

```
主色调：蓝色系
--blue-900: #1a2f52   --blue-800: #1e3a6e   --blue-700: #2d4a7a
--blue-600: #3a5a8c   --blue-500: #4a6fa0   --blue-400: #6a8cb8
--blue-300: #8aaad0   --blue-200: #b8cce0   --blue-100: #dce8f2

功能色：success #4a8c7a  /  warning #c4935a  /  danger #b85a5a
微信绿：--wechat-green: #07C160
```

### 2.2 设计原则

- **严禁使用emoji风格的icon**
- **严禁使用纯文字作为按钮**（按钮必须有明确的视觉样式）
- **严禁使用图片占位符** — 首页和房源详情必须展示真实房源照片
- 浮动按钮（float-btn）作为主要交互入口之一
- 底部浮标栏（float-bar）承载高频操作

### 3.3 主包页面功能详述

#### 3.3.1 登录页（pages/login）

**流程：**
1. 展示品牌页（Logo + 名称 + 标语）
2. 用户点击"微信授权登录"
3. 调用 `wx.login()` 获取code
4. 调后端 `/api/v1/auth/login` 换取JWT token
5. 调 `wx.getUserProfile()` 获取头像昵称（用户主动点击授权）
6. 更新用户信息到后端，存储token到本地
7. 跳转首页

**后端接口：**
- `POST /api/v1/auth/login` — 入参：code, encryptedData, iv；出参：access_token, refresh_token, user_info

#### 3.3.2 首页（pages/index）

**数据加载：**
- `GET /api/v1/cities` — 获取可选城市列表
- `GET /api/v1/market-stats?city_id=310000` — 行情数据（捡漏数/即将开拍数/昨日商家/昨日成交）
- `GET /api/v1/banners?city_id=310000` — 横幅列表
- `GET /api/v1/articles/recommend` — 法拍秘籍推荐文章（公众号文章）
- `GET /api/v1/properties/recommend?city_id=310000&page=1&page_size=6` — 精选房源

**交互：**
- 城市切换：点击城市选择器 → 底部弹出城市列表 → 切换后刷新全部数据
- 行情卡片点击 → 跳转对应筛选条件的房源列表
- 横幅点击 → 跳转公众号文章页
- 文章轮播点击 → 跳转公众号文章页
- 房源卡片点击 → 跳转房源详情
- 浮标（联系客服）→ 跳转联系客服页
- 浮标（返回顶部）→ 滚动到顶部

#### 3.3.3 房源列表（pages/property-list）

**筛选条件（来自甲方需求）：**
- 区域（全部区域、浦东新区、静安区、徐汇区、长宁区、黄浦区...）
- 起拍价（不限、100万以下、100-300万、300-500万、500万以上）
- 更多（拍品名称、法院评估价、建筑面积、拍卖状态、拍卖轮次、税费）

**排序方式：**
- 默认（综合排序）
- 起拍价（升序/降序）
- 评估价（升序/降序）
- 面积（升序/降序）

**接口：**
- `GET /api/v1/properties?city_id=310000&district=&price_min=&price_max=&keyword=&property_type=&auction_status=&auction_round=&sort_by=&sort_order=&page=1&page_size=20`

#### 3.3.4 房源详情（pages/property-detail）

**内容区块：**
1. 轮播图（真实房源照片，OSS CDN加载）
2. 标题 + 区域 + 收藏按钮
3. 价格四宫格：起拍价 / 评估价 / 折扣 / 保证金
4. 开拍倒计时/结束时间
5. Tab内容区：
   - **基本信息：** 小区名称、建筑面积、户型、楼层/总楼层、朝向、装修、物业类型、建筑年代、有无电梯、环线、板块
   - **法院公告：** 拍卖法院、案号、拍卖方式、加价幅度、咨询电话、看样时间
   - **小区详情：** 容积率、绿化率、物业费、停车位
   - **周边配套：** 地铁、学校、医院、商场、公园、银行
6. 底部浮标栏：分享 / 联系客服 / 板块分析（跳转公众号文章）/ 安全法拍（跳转公众号文章）

**接口：**
- `GET /api/v1/properties/{id}`

#### 3.3.5 公众号文章页（pages/article）

**内容来源：** "拍来盟科技"微信公众号文章，通过公众号接口或爬取获取

**我的服务宫格（3列 × 2行）：**

| 浏览记录 | 联系客服 | 邀请分享 |

**接口：**
- `GET /api/v1/user/profile` — 用户信息
- `GET /api/v1/user/stats` — 统计数据

### 4.3 功能模块详述

#### 4.3.1 权限管理
- 角色体系：**管理员 / 代理商 / 客户（普通用户）**

### B. 关键风险与应对

| 风险 | 应对 |
|------|------|
| 反爬升级导致抓取失败 | UA池+stealth JS+代理IP预留+熔断告警 |
