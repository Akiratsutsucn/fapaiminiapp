# 代理 IP 配置指南

## 为什么需要代理

- 服务器是腾讯云 IDC IP，**贝壳直接 CAPTCHA 拦截 `/xiaoqu/*` 路径**（家用 IP 没事，IDC 全死）
- 阿里司法拍卖 / 京东司法 / 公拍网在频繁访问时也会**触发短期封堵**
- 当前已用 cookie + UA 轮换 + 失败重试，**还是建议补上代理 IP** 把成功率从 ~70% 提到 ~95%+

## 推荐方案：住宅代理（家庭宽带 IP）

国内常见服务商（按个人开发者性价比排序）：

| 服务商 | 月费 | 流量 | 备注 |
|---|---|---|---|
| **快代理** | ~50 元 | 1 GB | 国内住宅，支持 HTTP/SOCKS5。https://www.kuaidaili.com |
| **蘑菇代理** | ~60 元 | 1.5 GB | 同上 |
| **芝麻代理** | ~80 元 | 1 GB | 老牌，可选独享 |
| **巨量代理** | ~100 元起 | 流量计费 | 专业级 |

### 不推荐
- ❌ **免费代理 / 公开代理列表** — 99% 已死，剩 1% 速度奇慢，且爬虫日志会被代理商记录
- ❌ **机房代理（IDC）** — 跟你现在服务器一样会被识别为爬虫
- ❌ **国外住宅代理（Bright Data / Oxylabs）** — 顶级但贵 $300+/月

## 如何配置

买好后会拿到几个代理 URL，比如：

```
http://kdluser123:abc123@123.45.67.89:8888
http://kdluser123:abc123@98.76.54.32:8888
socks5://user:pass@10.20.30.40:1080
```

### 1. 写入服务器 `.env`

```bash
ssh -i xiaochengxu.pem ubuntu@122.51.156.252

# 编辑 .env，把多个代理用逗号分隔
sudo vim /opt/fapai/.env

# 末尾加一行：
PROXY_POOL=http://user:pass@1.2.3.4:8888,http://user:pass@5.6.7.8:8888,socks5://10.20.30.40:1080
```

### 2. 重启服务

```bash
sudo systemctl restart fapai-backend
```

### 3. 测试

```bash
cd /opt/fapai && sudo -u www-data /opt/fapai/venv/bin/python -m crawler.community_scraper --city 上海 --limit 5 --force
```

**期待看到**：
- 失败时日志提示 `[anti] proxy 标记失效` → 自动切下一个
- 风控时 `[anti] host xxx 触发风控冷却 30min`

## 代理池工作原理

`crawler/anti_block.py` 实现：

1. **轮换策略**：每次请求挑一个代理
2. **失败标记**：连接超时 / 返回风控 → 该代理冷却 15 分钟
3. **平台冷却**：某 host 多次失败 → 整个 host 冷却 30 分钟（保护代理 IP 不被加入贝壳黑名单）
4. **优雅降级**：所有代理都坏 → 自动降级直连（仍带 UA 轮换 + 重试）

## 流量预估

- 三平台日常爬取（每天凌晨 2 点）：~5 MB
- 贝壳小区抓取（一次回填约 100 个小区）：~3 MB
- **每月总流量**：约 **300 MB**

也就是说**最低档 50 元/月套餐绰绰有余**。

## 不用代理也能跑吗？

可以。当前架构（cookie + UA 轮换 + 重试 + 风控感知冷却）已经能覆盖：
- ✅ 阿里、京东、公拍网（这三家反爬不算极端）
- ⚠️ 贝壳的搜索 sug API（直连可用）
- ❌ 贝壳的小区详情页（IDC IP 100% 被拦，必须代理）

**当前贝壳详情字段抓不到**就是这个原因。所以代理主要解决贝壳详情页问题。
