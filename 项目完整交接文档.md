## 2. 小程序前端（`miniprogram/`）

### 2.1 构建方式（重要）
- **原生微信小程序 + TypeScript，无 npm 构建**（无 `package.json`、无 `miniprogram_npm`）。
- TS 源文件与编译产物 `.js` **同名并存**（如 `house-card.ts` + `house-card.js`）。开发者工具自动编译 TS→JS。
- **改代码时 `.ts` 和 `.js` 必须同步修改**（本会话一直如此做）——只改 `.ts` 不改 `.js`，线上跑的是 `.js` 不会生效。
- `tsconfig.json`：target ES2018，module commonjs，outDir 同目录。
- `project.config.json`：`libVersion: 3.15.2`，`appid: wx4ed176b1fd0eac4f`。

### 3.6 关键业务逻辑（单一事实源，务必复用，勿各写各的）
- **`app/core/auction_status.py` — 拍卖状态实时计算**（核心！）：
  - 不信任库里 `auction_status` 快照（抓取时刻可能已过期），统一按 `start_time/end_time + 当前时间` 实时推导。
  - `effective_status()`(纯Python) 与 `effective_status_sql()`(SQL case 表达式,用于 WHERE/ORDER BY) 两版必须共用。
  - 结果态 `RESULT_STATES`(已成交/已撤回/中止/流拍)保留原值；时序态(即将开拍/进行中/已结束)按时间算。
  - `MOBILE_VISIBLE_STATUSES`=可参拍(即将开拍/进行中)；无可参拍时降级展示 `MOBILE_FALLBACK_STATUSES`(已成交)，避免页面空白。
  - **昨日上架口径**：按 `publish_date`(法院落款日)，**排除京东**(`UNRELIABLE_PUBLISH_DATE_PLATFORMS`,数据不可信)；首页统计与列表入口共用此函数。
  - **捡漏定义**：折扣率(起拍/评估) ∈ [0.1, 0.65]（1折~6.5折）且可参拍。
- **`app/core/district_priority.py` — 主城区优先排序**：4 档（主城+一拍二拍 / 郊区+一拍二拍 / 主城+变卖 / 郊区+变卖）。`tier_sql_expr()` 用于 ORDER BY。主城清单是业务定义（可调）。
- **`app/api/v1/properties.py` 列表封面图**（本会话新增）：`_list_cover(p)` 优先 `thumb_url`→回退 `image_url`，过滤 hidden=1；`_normalize_img()` 把 `http://裸IP/images/...` 重写为 `https://xcxapi.fapaizhelianmeng.cn/images/...`（小程序强制 https，否则加载不出）。
- **默认排序**：`tier_sql_expr().asc()` → 开拍时间 NULL 排最后 → 开拍时间 asc → created_at desc。

## 4. 数据抓取爬虫（`crawler/`）

### 4.1 整体架构
- 入口 `crawler/main.py`，命令行：
  ```bash
  python -m crawler.main                          # 全量
  python -m crawler.main --source taobao|jd|gpai  # 指定平台
  python -m crawler.main --city 上海|宁波|杭州     # 指定城市
  python -m crawler.main --max-pages 2 --force-refresh --log-level DEBUG
  python -m crawler.main --schedule               # 定时模式
  ```
  无 `--task-id` 时自动 `create_auto_task()` 记录后台任务。
- 核心引擎 `engine.py` 的 `CrawlEngine.run()`：加载配置 → 按平台分组 → 逐平台爬取 → 增量对比 → 汇总日志。
- 平台工厂 `PLATFORM_FACTORY`（engine.py:60）：
  ```
  阿里拍卖 → (TaobaoPaiMaiCrawler, TaobaoPaiMaiDetailParser)
  京东拍卖 → (JDAuctionCrawler, JDDetailParser)
  公拍网   → (GPaiCrawler, GPaiDetailParser)
  ```
- 单平台主流程 `_process_platform`：collect_list_items（列表）→ should_fetch（去重：内存+DB+stale）→ process_detail（并发详情→解析→图片→入库）→ 后处理（社区补全/地理编码/POI/贝壳/状态自校正/文本补全/智能标签）。
- **增量对比 `_run_incremental_comparison`**（engine.py:904）：当次抓到的 URL 集合 vs DB；DB 中"即将开拍/进行中"但本次未见的标记为"已结束"；统计 relisted（重新上架）。

### 4.2 三平台抓取
- **阿里拍卖** `platforms/taobao_paimai.py`：
  - MTOP 接口（`h5api.m.taobao.com`），MD5 签名 `token&t&appKey&data`；从 cookie 取 `_m_h5_tk`。
  - 上海直连列表 API，宁波/杭州用 keyword 参数过滤；诉讼资产 `zcBizTypes=["6"]`。
  - **面积**：详情 `auctionHouse.buildingArea`，**阿里返回值已是 m²，无需 /100**（注意：旧注释里曾有 /100 转换的说法，以实际为准）。
  - 详情 API 失败时 Playwright HTML 回退。并发=1 防限流。
  - Cookie：`settings.TAOBAO_COOKIE`。
- **京东拍卖** `platforms/jd.py`：
  - React SPA，列表 JSON 需页面 JS 生成 h5st 签名 → 用 **Playwright 拦截 `paimai_unifiedSearch` 响应**，翻页点"下一页"让页面重新签名。
  - **每城市独立 browser context**（修复宁波/杭州抓取为 0 的关键，避免风控污染）。详情 `getProductBasicInfo` 无需签名，httpx 直连。
  - ⚠️ 京东数据质量差：publish_date 不可信（昨日上架口径已排除京东）、起拍价/面积/长 id 曾有 bug（见第 7 章）。
- **公拍网** `platforms/gpai.py`：
  - 搜索端点已下线 → 改为抓 `www.gpai.net/sf/` 首页 item2.do 链接 + 从 max_id 起递增扫 200 个新 ID。
  - `s.gpai.net` 封服务器 IP，需 `GPAI_PROXY` 住宅代理；`www.gpai.net` 详情页服务器直连。资产类型 at=376+381，城市 cityNum=31/3302/3301。
  - **法院公告落款日期提取** `parsers/gpai_detail.py`：正则支持阿拉伯+中文数字（二〇二六年五月十一日），取最早日期作为 `publish_date`（真实上架日）。

### 4.3 城市与过滤逻辑（engine.py，本会话重点修复区）
- `CITY_ID_MAP`：上海 310000 / 宁波 330200 / 杭州 330100。`VALID_DISTRICTS`：三市合法辖区白名单。
- **外省同名楼盘过滤 `_re_other_province`**（engine.py:44）：标题/地址开头是外省省名或市县名（即使后含"上海/杭州/宁波"也判外省）。
  - **本会话扩充**：补了 昆明/绍兴/启东/江阴/青岛/十堰/蒙自/滨州/阜阳/荆州/武威/黄山/北海/南充/东海/张家口/临邑/微山/盘州/库尔勒/日照/莱西/东明/清河/奇台/道真/安乡/安化/云岩/郏县 等市县名。
  - **本会话关键修复**：原来只 match `address` 开头，导致"江阴市上海花园""绍兴市上虞区新上海花园"因标题含"上海"被错判入库 → 改为**同时 match title 与 address 开头**。

### 4.4 反爬机制
- **SOCKS 桥接**：`fapai-socks-bridge.service` 把本地无认证 socks5 → 云镜认证 socks5 住宅 IP，阿里抓取走住宅 IP 绕淘宝限流。
- **隧道代理换 IP** `tunnel_proxy.py`（90daili）：风控时 `trigger_ip_change()`，节流间隔 `TUNNEL_IP_CHANGE_MIN_INTERVAL=30s`。
- **URL 冷却** `anti_block.py`：`check_url_cooldown` / `mark_url_blocked`（命中触发 30 分钟冷却）；连续失败阈值 10。
- **智能重试** `http_get_with_retry`：5xx 线性退避重试；429/403 切代理+长退避；最多 3 次。alicdn 图片对机房 IP 返 HTTP 420 → 直连为主、多次失败走代理兜底。
- **代理池** `ProxyPool`：轮换+失败标记，失败代理 15 分钟冷却，优先直连。

### 4.5 入库与图片
- `storage/repository.py`：`PropertyRepository.upsert()` 按 `source_url` 判断 INSERT/UPDATE；`PropertyImageRepository.batch_upsert()` 先删旧图再批量插。
- 图片 `pipelines/image_processor.py`：下载（`_fetch_bytes` 4 次重试，420 加长退避，兜底代理）→ `transform_image_url`（京东 /s800x800、阿里去尺寸标记）→ `_resize_to_webp`（max_width 1080, quality 80）→ 同步生成缩略图（max_width 300）。
- 本地存储 `pipelines/local_storage.py`：`/picture/properties/{property_id}/{md5前12}_{序号}_{full|thumb}.webp`。
- **垃圾图过滤**：二维码/banner/logo/纯色 → `hidden=1` + `hide_reason`，后台可恢复。

### 4.6 爬虫数据库配置 `storage/db.py`
- 独立于后端但同库：`pool_size=5, max_overflow=10, pool_pre_ping=False, pool_recycle=3600`。
- ⚠️ **历史坑**：曾因 MySQL 连接中断后 session 未回滚导致级联失败（爬虫长跑 10+ 小时崩溃），已修（见第 7 章）。

### 4.7 配置 `config.py` 关键项
- `STALE_REFRESH_HOURS=24`（24h 后重抓已有记录）、`STALE_DAYS_THRESHOLD=7`（7 天未见标记结束）、`DAILY_CRON="0 3 * * *"`、`DETAIL_PAGE_CONCURRENCY=3`、图片 `IMAGE_MAX_WIDTH=1080`/`THUMB_WIDTH=300`/`WEBP_QUALITY=80`。
- 代理：`PROXY_URL`/`GPAI_PROXY`/`IMAGE_PROXY`、`TAOBAO_COOKIE`、`TUNNEL_IP_CHANGE_URL`。
- 日志：`crawler/logs/crawler_{日期}.log`（DEBUG，30天）+ `crawler_error_{日期}.log`（ERROR）。

## 7. 已知坑与根因（踩过的雷，避免重蹈）

1. **小程序"请求失败"随机高发** → 根因：后端连接池 `pool_pre_ping=False` 且无 `pool_recycle`，MySQL `wait_timeout` 掐断空闲连接后，陈旧连接被取用报 2013 Lost connection。修复：`pool_recycle=3600`（后端 + 爬虫两处）。`pre_ping` 因 aiomysql ping() 签名不兼容**不能用**。

2. **外省同名楼盘混入**（"江阴市上海花园""绍兴市上虞区新上海花园"被当成上海房源）→ 根因：城市判定 `_re_other_province` 原来只 match address 开头，而 address 被搜索城市兜底覆盖，真实外地信息只在 title 里。修复：同时 match title 与 address 开头 + 扩充外省市县名单。**反向陷阱**：蒙自路/河南南路/山东中路/清河湾路是上海真实路名，清洗时勿误删——必须逐条核对，删除前先备份（本会话用 `deleted_backup_foreign.json` 备份了 62 条）。

3. **爬虫长跑（10+ 小时）崩溃** → 根因：MySQL 连接中断后 session 未回滚导致后续查询级联失败。已修（commit d784502）。

4. **京东数据质量差**：起拍价/增价幅度/面积曾整列缺失（长 id 根因，a89453e 修）；publish_date 不可信 → 昨日上架口径**排除京东**；成交单价存在单位错乱（>100万/㎡ 或 <1000/㎡）→ 后端用 `DEAL_UNIT_PRICE_MIN/MAX`=[1000, 300000] 过滤脏数据。

5. **京东多城市抓取为 0**（宁波/杭州）→ 根因：React SPA 列表需 h5st 签名，且城市间风控互相污染。修复：Playwright 拦截响应 + **每城市独立 browser context**。

6. **拍卖状态过期导致上海可参拍房源被全筛掉** → 根因：信任库里 `auction_status` 文本快照。修复：改 `effective_status()` 按时间实时计算，全后端统一复用（083f69f）。

7. **公拍网面积解析错误**（误抓评估价/漏抓）→ 已修（862391a）。`s.gpai.net` 封服务器 IP，详情页要用 `www.gpai.net` 直连 + 列表走住宅代理。

8. **小区名提取错误**（从整页 text 误抓）→ 改为只从当前标的 title/address 提取（新增 extract_community_from_title），修正存量 823 条（ff0f8c2）。

9. **面积缺失 17.6%**（本会话排查，**非 bug**）：可见房源 274 条无面积，其中 97 条是车位/车库（本就无标准建面），159 条住宅是阿里源头 `hArea`/`buildingArea` 字段就没返回、description 也无面积文本可提取。属源头缺失，无法在后端补。

10. **缩略图 thumb_url 存的是 `http://裸IP`** → 小程序强制 https+备案域名加载不出（部分卡片封面空白）。修复：`_normalize_img()` 重写为 https 域名。thumb 覆盖率约 83%。

11. **lazyCodeLoading 渲染错误风险**（本会话专门评估）：按需注入只注入静态可分析的组件，对 抽象节点/generic、selectComponent 动态引用、componentPlaceholder、custom-tab-bar(低版本) 有漏注风险。本项目逐项排查**全不命中**（基础库 3.15.2、组件全静态引用、custom-tab-bar 四件套齐全），安全。改注入级配置后**务必真机验证**（模拟器与真机注入行为可能不同）。

12. **TS/JS 双写**：小程序无 npm 构建，`.ts` 与 `.js` 并存，改一个必须改另一个，否则线上（跑 .js）不生效。

13. **线上 ≠ git**：`/opt/fapai/` 是独立运行副本，靠 scp 手动同步。"代码已 commit 但线上没变"或反之，都因为漏了 scp/restart。
