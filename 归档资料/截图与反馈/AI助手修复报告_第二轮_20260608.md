## 问题分析

### 问题1：删除会话时sessionId为undefined

**错误日志：**
```
api/admin/ai/sessions/undefined:1  Failed to load resource: the server responded with a status of 404 ()
```
（重复5次）

**根本原因：**
1. 会话列表中存在 `id` 字段为 `undefined` 或 `null` 的session对象
2. 用户点击删除按钮时，传入了无效的sessionId
3. 第一次修复只在 `onSelectSession` 中添加了校验，但 `onDeleteSession` 和 `loadSessions` 没有防御

### 问题2：TextArea的'in'操作符错误

**错误日志：**
```
TypeError: Cannot use 'in' operator to search for 'key' in 
TypeError: Cannot use 'in' operator to search for 'key' in 测试
TypeError: Cannot use 'in' operator to search for 'key' in 查看用户增长情况
```
（重复12次）

**根本原因：**
1. 第一次修复使用 `@input="handleInput"` 试图拦截事件，但无效
2. 错误来自 TDesign 组件内部对 `v-model` 的处理逻辑
3. `v-model` 双向绑定触发了组件内部的 `'key' in value` 检查
4. 必须从根源解决：移除 `v-model`，改用单向绑定 + 事件监听

## 第二次修复方案

### 修复1：移除v-model，使用单向绑定

**文件：** `admin-web/src/views/ai/AiAssistantView.vue` (第80-87行)

**修改前：**
```vue
<t-textarea
  v-model="inputMessage"
  placeholder="输入您的问题..."
  :autosize="{ minRows: 2, maxRows: 6 }"
  @keydown.enter.ctrl.prevent="onSendMessage"
  :disabled="streaming"
  @input="handleInput"
/>
```

**修改后：**
```vue
<t-textarea
  :value="inputMessage"
  placeholder="输入您的问题..."
  :autosize="{ minRows: 2, maxRows: 6 }"
  @keydown.enter.ctrl.prevent="onSendMessage"
  @change="handleTextareaChange"
  :disabled="streaming"
/>
```

**关键改动：**
- `v-model="inputMessage"` → `:value="inputMessage"` （单向绑定）
- `@input="handleInput"` → `@change="handleTextareaChange"` （手动更新）
- 移除了 `@input` 事件，避免触发TDesign内部的类型检查

### 修复2：更新事件处理函数

**修改前：**
```javascript
function handleInput(value: string) {
  if (typeof value === 'string') {
    inputMessage.value = value
  }
}
```

**修改后：**
```javascript
function handleTextareaChange(value: string) {
  // 使用 change 事件代替 v-model，避免 TDesign 组件内部的 'in' 操作符错误
  inputMessage.value = value || ''
}
```

### 修复3：删除会话的ID校验

**添加：**
```javascript
async function onDeleteSession(sessionId: number) {
  if (!sessionId) {
    console.error('删除会话失败：会话ID无效', sessionId)
    MessagePlugin.error('会话ID无效')
    return
  }
  // ... 其余代码
}
```

### 修复4：会话列表数据过滤

**修改后：**
```javascript
async function loadSessions() {
  try {
    const data = await listAiSessions()
    // 过滤掉无效的会话（id为空或undefined）
    sessions.value = data.filter(s => s && s.id)
    // ...
  }
}
```

## 技术总结

### 为什么第一次修复不够

1. **防御不全面** - 只在 `onSelectSession` 加了校验，但数据源和删除逻辑没有保护
2. **治标不治本** - 试图拦截 `@input` 事件，但TDesign的类型检查发生在更早的阶段
3. **没有理解根因** - 'in' 操作符错误是 `v-model` 触发的，不是输入值的问题

### 第二次修复的关键

1. **从源头防御** - 在 `loadSessions` 就过滤无效数据
2. **全链路校验** - 选择、删除、加载三个地方都加了ID校验
3. **绕过问题组件逻辑** - 不用 `v-model`，完全避开TDesign内部的类型检查
4. **使用单向数据流** - `:value` + `@change` 手动控制，更可控
