## 问题1：会话ID为undefined导致404错误

### 错误表现
```
api/admin/ai/sessions/undefined/messages: 404
加载消息失败: AxiosError: Request failed with status code 404
```

### 根本原因
1. 用户首次访问AI助手或删除所有会话后，`currentSessionId` 为 `null`
2. 页面在没有会话的情况下仍然尝试加载消息
3. API请求使用 `undefined` 作为会话ID

### 修复方案
**文件：** `admin-web/src/views/ai/AiAssistantView.vue`

#### 修改1：自动创建首个会话（第127-137行）
```javascript
async function loadSessions() {
  try {
    const data = await listAiSessions()
    sessions.value = data
    if (data.length > 0 && !currentSessionId.value) {
      onSelectSession(data[0].id)
    } else if (data.length === 0) {
      // 如果没有会话，自动创建一个
      await onCreateSession()
    }
  } catch (err) {
    console.error('加载会话列表失败:', err)
  }
}
```

#### 修改2：会话ID校验（第150-164行）
```javascript
async function onSelectSession(sessionId: number) {
  if (!sessionId) {
    console.warn('会话ID无效，跳过加载')
    return
  }
  currentSessionId.value = sessionId
  try {
    const data = await getSessionMessages(sessionId)
    messages.value = data
    scrollToBottom()
  } catch (err) {
    console.error('加载消息失败:', err)
    messages.value = []
  }
}
```

#### 修改3：删除会话后自动创建（第162-180行）
```javascript
async function onDeleteSession(sessionId: number) {
  try {
    await deleteAiSession(sessionId)
    sessions.value = sessions.value.filter(s => s.id !== sessionId)
    if (currentSessionId.value === sessionId) {
      if (sessions.value.length > 0) {
        onSelectSession(sessions.value[0].id)
      } else {
        // 删除最后一个会话后，创建新会话
        currentSessionId.value = null
        messages.value = []
        await onCreateSession()
      }
    }
    MessagePlugin.success('已删除对话')
  } catch (err) {
    MessagePlugin.error('删除对话失败')
  }
}
```

## 问题2：TextArea输入框 'in' 操作符类型错误

### 错误表现
```
TypeError: Cannot use 'in' operator to search for 'key' in 
    at r.<computed>.r.<computed> (index-CpjPs9Yu.js:18:11918)
    at HTMLTextAreaElement.r (index-CpjPs9Yu.js:18:7825)
```

### 根本原因
TDesign Vue Next 的 `t-textarea` 组件在处理 `v-model` 时，内部对事件对象进行类型检查。当值为空字符串或其他非对象类型时，组件可能尝试用 `'key' in value` 进行检查，导致类型错误。

### 修复方案
**文件：** `admin-web/src/views/ai/AiAssistantView.vue`

#### 修改1：添加 input 事件处理器（第79-88行）
```vue
<t-textarea
  v-model="inputMessage"
  placeholder="输入您的问题..."
  :autosize="{ minRows: 2, maxRows: 6 }"
  @keydown.enter.ctrl.prevent="onSendMessage"
  :disabled="streaming"
  @input="handleInput"
/>
```

#### 修改2：输入值类型校验（第254-268行）
```javascript
function handleInput(value: string) {
  // 确保输入值是字符串类型，防止 TDesign 组件内部的类型检查错误
  if (typeof value === 'string') {
    inputMessage.value = value
  }
}
```

#### 修改3：防止默认回车行为
将 `@keydown.enter.ctrl` 改为 `@keydown.enter.ctrl.prevent`，避免事件传播问题
