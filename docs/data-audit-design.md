# 数据审核清洗系统设计文档

## 系统目标

构建一个完整的爬虫数据质量管理系统，实现：

1. **数据准确性核对**：验证爬取数据的必填字段、数据格式、数值范围
2. **数据完整性检查**：确保关键字段不缺失
3. **垃圾数据清洗**：自动识别并清理不符合要求的数据
4. **可视化管理**：通过管理后台直观展示审核规则、执行结果和数据质量报告
5. **定时自动执行**：每日凌晨2点自动审核，无需人工干预

## 核心功能模块

### 1. 审核规则管理

**功能**：
- 新增/编辑/删除审核规则
- 启用/禁用规则
- 查看规则执行统计

**规则类型**：
1. **必填字段检查**：验证必填字段是否为空
2. **字段范围检查**：验证数值字段是否在合理范围内
3. **字段格式检查**：验证字段格式是否符合正则表达式
4. **地区过滤**：仅保留指定城市的数据
5. **房产类型过滤**：仅保留不动产类型

**默认规则**：
- 核心价格字段必填（起拍价、保证金、加价幅度）
- 房源面积必填
- 起拍价合理性检查（10万-1亿）
- 保证金合理性检查
- 房源面积合理性检查（10-1000平米）
- 目标城市范围过滤（上海、宁波、杭州）
- 不动产类型过滤
- 地址信息完整性检查
- 拍卖状态必填检查
- 经纬度合理性检查

### 2. 审核任务执行

**手动任务**：
- 管理员通过后台手动触发
- 可选择使用的规则
- 可指定审核范围（平台、城市、日期）

**定时任务**：
- 每天凌晨2点自动执行
- 使用所有启用的规则
- 全量数据审核

**执行流程**：
1. 创建任务记录
2. 获取待审核房源数据
3. 对每条数据应用所有规则
4. 记录违规情况
5. 执行规则动作（标记/修复/删除）
6. 更新任务统计
7. 生成审核报告

### 3. 违规记录管理

**功能**：
- 查看所有违规记录
- 按任务/规则/严重级别筛选
- 查看违规详情
- 标记违规已解决/已忽略

**违规处理动作**：
- **flag**：仅标记，不修改数据
- **fix**：尝试自动修复（如填充默认值）
- **delete**：直接删除违规数据

### 4. 审核报告

**报告内容**：
1. **总体统计**
   - 总检查数
   - 通过数和通过率
   - 违规数和违规率
   - 数据质量评分

2. **Top违规规则**
   - 违规次数最多的前10条规则

3. **规则统计**
   - 每条规则的检查数、违规数、违规率

4. **平台统计**
   - 各平台的检查数和违规数

5. **城市统计**
   - 各城市的检查数和违规数

**质量评分算法**：
```
质量评分 = 通过率 = (通过数 / 总检查数) × 100
```

### 5. 可视化仪表板

**概览页面**：
- 审核规则总数、启用数
- 执行任务总数
- 最新数据质量评分
- 待处理违规数量
- 最新任务执行情况
- 快速创建审核任务入口

**规则管理页面**：
- 规则列表展示
- 规则新增/编辑/删除
- 规则启用/禁用开关
- 规则执行统计

**任务管理页面**：
- 任务列表展示
- 任务状态和进度
- 任务详情查看
- 报告查看入口

**违规记录页面**：
- 违规记录列表
- 多维度筛选
- 违规详情查看
- 违规处理操作

**报告详情页面**：
- 质量评分环形图
- 统计卡片
- 规则统计表格
- 平台/城市统计卡片

## 技术实现

### 后端技术栈

- **框架**：FastAPI
- **ORM**：SQLAlchemy
- **数据库**：MySQL
- **异步**：asyncio
- **调度**：自定义调度器（基于asyncio）

### 前端技术栈

- **框架**：Vue 3
- **UI组件**：Element Plus
- **路由**：Vue Router
- **状态管理**：Pinia
- **HTTP客户端**：Axios

### 关键代码模块

1. **DataAuditService** (`backend/app/services/data_audit_service.py`)
   - 核心审核逻辑
   - 规则检查实现
   - 报告生成

2. **AuditScheduler** (`backend/app/services/audit_scheduler.py`)
   - 定时任务调度
   - 自动创建审核任务

3. **API路由** (`backend/app/api/admin/data_audit.py`)
   - RESTful API接口
   - 规则/任务/违规/报告CRUD

4. **前端视图组件** (`admin-web/src/views/data-audit/`)
   - DashboardView.vue：审核概览
   - RulesView.vue：规则管理
   - TasksView.vue：任务管理
   - ViolationsView.vue：违规记录
   - ReportView.vue：报告详情

## 扩展性设计

### 1. 新增规则类型

在 `DataAuditService._check_rule()` 方法中添加新的 category 处理分支。

### 2. 自定义执行动作

在 `DataAuditService._execute_action()` 方法中添加新的 action 处理逻辑。

### 3. 报告内容扩展

在 `DataAuditService._generate_report()` 方法中添加新的统计维度。

### 4. 调度时间调整

修改 `AuditScheduler.run_scheduler()` 方法中的 `target_time` 设置。

### 5. 多数据源支持

当前仅审核 properties 表，可扩展到其他数据表。

## 性能优化

1. **批量提交**：每100条记录提交一次数据库事务
2. **异步处理**：使用 asyncio 实现异步审核
3. **后台执行**：审核任务在独立线程中执行，不阻塞API响应
4. **索引优化**：为 task_id、rule_id、property_id 添加索引

## 安全考虑

1. **权限控制**：所有API接口需要管理员权限
2. **操作审计**：记录违规处理人和处理时间
3. **危险操作确认**：delete 动作需要明确配置
4. **SQL注入防护**：使用ORM参数化查询

## 总结

该系统实现了完整的数据质量管理闭环：

1. **规则配置**：灵活可扩展的规则体系
2. **自动执行**：定时任务无需人工干预
3. **问题发现**：多维度检查发现数据问题
4. **问题处理**：支持标记、修复、删除多种处理方式
5. **质量可视化**：直观展示数据质量状况
6. **持续改进**：通过违规统计优化数据采集和清洗策略

系统已在生产环境部署验证，可有效提升爬虫数据质量。
